Resource Profile: Location

Official URL: http://cuida.pruebas.dherly.jurisdiction/fhir/laboratory/StructureDefinition/LocationOH Version: 0.0.5
Draft as of 2024-06-28 Computable Name: LocationOH

Copyright/Legal: UMA Health 2024

Representa ubicaciones físicas en el Ecosistema de Salud, con especial énfasis en Establecimientos de Salud (IPRESS). Se utiliza para estandarizar su identificación, localización y características principales.

Perfil Location: Representación de Establecimientos de Salud (IPRESS) en el Ecosistema de Salud

1. Funcionalidad Principal:

  • Estandariza la representación de IPRESS registradas en SUSALUD para el Ecosistema de Salud.
  • Asegura la identificación del código único de IPRESS registrado en el sistema de RENIPRESS (Registro Nacional de Instituciones Prestadoras de Servicios de Salud).
  • Define la información clave del establecimiento: nombre, tipo físico, estado operativo, dirección (con extensiones para UBIGEO) y organización gestora.
  • Soporta la definición de relaciones jerárquicas entre componentes de una IPRESS (ej. un consultorio dentro de un hospital) mediante el elemento partOf.

2. Elementos Esenciales

Los siguientes elementos deben estar presentes en cada instancia de Location:

  • identifier (Identificador Único de IPRESS): Código único de IPRESS del establecimiento registrado en RENIPRESS.
    • identifier.use: Fijo a #official.
    • identifier.system: Fijo a la URI de SUSALUD (https://fhir.onehealth.com.pe/susalud).
    • identifier.value: Contiene el código único de IPRESS registrado en REIPRESS.
    • identifier.assigner: Referencia a la Organization correspondiente.
  • status (Estado del Registro): Estado actual del registro de la IPRESS (ej. active, inactive).
  • name (Nombre de la IPRESS): Nombre oficial o descriptivo del establecimiento.
  • managingOrganization (Organización Gestora): Referencia a la Organization que administra o es propietaria de la IPRESS.

3. Elementos de Soporte Obligatorio (Must Support - MS)

Los sistemas que conformen con este perfil deben ser capaces de procesar y persistir (soportar) los siguientes elementos si están presentes:

  • address (Dirección): Dirección física detallada del establecimiento.
    • Soporte obligatorio para las extensiones de UBIGEO: Extension-Department, Extension-Province, Extension-District.

4. Lógica de Identificadores (IPRESS) y Estrategia de Búsqueda

  • El elemento identifier en Location es mandatorio y se utiliza exclusivamente para el código único de IPRESS del establecimiento de salud.
  • El sub-elemento identifier.system está fijo al valor https://fhir.onehealth.com.pe/susalud, garantizando la referencia al registro nacional de SUSALUD.
  • La búsqueda de estos establecimientos se realiza utilizando este identificador específico, consultando por identifier con el formato https://fhir.onehealth.com.pe/susalud|[CODIGO_IPRESS].

5. Extensiones

El perfil Location utiliza las siguientes extensiones para capturar información geolocalizada detallada, esencial en el contexto peruano:

  • Para Location.address.extension:
    • Extension-Department: Código UBIGEO del departamento donde se ubica la IPRESS.
    • Extension-Province: Código UBIGEO de la provincia donde se ubica la IPRESS.
    • Extension-District: Código UBIGEO del distrito donde se ubica la IPRESS.

Este perfil es crucial para la correcta referenciación, geolocalización y gestión de los Establecimientos Prestadores de Servicios de Salud (IPRESS) donde se proveen servicios de salud o donde ocurren eventos relevantes para el Ecosistema de Salud.
